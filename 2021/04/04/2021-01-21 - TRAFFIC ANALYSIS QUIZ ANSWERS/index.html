NOTES:

I wrote a Wireshark tutorial about examining Ursnif infections that was published in December 2019:

  • • https://unit42.paloaltonetworks.com/wireshark-tutorial-examining-ursnif-infections/

It still applies to the Ursnif infections I run across today. For this exercise, we find the following alerts:

  • • 11 different alert titles for traffic to 209.141.51.196\ over TCP port 80. These alerts hint that an “evil EXE” was downloaded from that IP address over unencrypted HTTP traffic.
  • • 1 alert for a JA3 hash that’s possibly Gozi.
  • • 1 alert for DNS to myip.opendns.com\ for an IP address check.
  • • 1 alert for traffic to 162.0.224.165\ that triggered on a url ending with .rar.
  • • 2 alerts for malicious SSL certificates associated with Ursnif or Zeus Panda Banker

![image-20210404073218050](/Users/yueting/Library/Application Support/typora-user-images/image-20210404073218050.png)

Shown above: The alerts from this exercise, and their associated IP addresses.

Based on my experience:

  • • I haven’t personally seen any active samples of Zeus Panda Banker in the past two years or so, but I have seen a lot of Ursnif (also called Gozi or ISFB). 我个人在过去两年间没有看到Zeus Panda Banker ，但看到了大量的Ursnif (also called Gozi or ISFB)
  • • Ursnif (Gozi/ISFB) infections I’ve generated always cause a DNS query to myip.opendns.com. Ursnif (Gozi/ISFB)感染造成了DNS查询对于myip.opendns.com
  • • For Ursnif, additional Ursnif components, follow-up malware, or other data are always retrieved using HTTP GET requests ending in .rar. These binaries are encoded or otherwise encrypted when sent over the web and decrypted/decoded on the infected host. 对于Ursnif，附加的Ursnif组件，后续的恶意代码或者其他数据经常使用HTTP GET 以.rar\结尾检索。二进制被编码或加密当发送到web在受感染主机上解码或解密。
  • • 72.21.81.200\ and the associated domains (iecvlist.microsoft.com\ and ctldl.windowsupdate.com) are not malicious, so the ET JA3 Hash alert for possible Gozi is a false positive.

There was a malware DLL returned from http://209.141.51.196/files/1.bin\. Unfortunately, you cannot export the above file from the pcap using the File –> Export Objects –> HTTP\ menu path like I’ve mentioned in the Wireshark tutorial about exporting objects from a pcap (see the links section at the beginning of this document).
