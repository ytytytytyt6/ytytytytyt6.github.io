2019-01-28 - TRAFFIC ANALYSIS EXERCISE - TIMBERSHADE

SCENARIO

LAN segment data:

  • LAN segment range: 172.17.8.0/24 (172.17.8.0 through 172.17.8.255)
  • Domain: timbershade.info
  • Domain controller: 172.17.8.2 - Timbershade-DC
  • LAN segment gateway: 172.17.8.1
  • LAN segment broadcast address: 172.17.8.255

YOUR TASK

Answer the following questions:

  • What is the IP address of the infected Windows host?
  • What is the MAC address of the infected Windows host
  • What is the host name of the infected Windows host
  • What is the Windows user account name for the infected Windows host
  • What is the SHA256 file hash of the Windows executable file sent to the infected Windows host?
  • Based on the IDS alerts, what type of infection is this?

解析

如果您在Wireshark中查看pcap，请使用“统计”菜单选项进行“对话”，如下图所示。
![image-20210404001359954](/Users/yueting/Library/Application Support/typora-user-images/image-20210404001359954.png)
上图所示：统计信息→会话
我们已经知道LAN数据用于172.17.8.0/24地址空间，并且仅列出了一个IP地址，它不是域控制器（172.17.8.2）或广播地址（172.17.8.255），即172.17.8.109。此外，警报文件仅显示172.17.8.109的警报。

![image-20210404001500901](/Users/yueting/Library/Application Support/typora-user-images/image-20210404001500901.png)上图所示：寻找172.17.8.0/24地址。

您可以通过查看DHCP流量（过滤：bootp）或NBNS流量（过滤：nbns），将IP地址与MAC地址和主机名相关联，如下图所示。

上图所示：筛选DHCP流量（bootp）以查找主机名。
![image-20210404001712784](/Users/yueting/Library/Application Support/typora-user-images/image-20210404001712784.png)
上图所示：过滤NBNS流量以查找主机名。
要查找Windows用户帐户名，请使用以下Wireshark筛选器：
kerberos.CNameString and !(kerberos.CNameString contains $)
注意：您应该使用最新版本的Wireshark来执行此操作（我认为是2.2版和更高版本）。然后继续工作，在帧详细信息中找到CNameString值，如下图所示。
![image-20210404001909534](/Users/yueting/Library/Application Support/typora-user-images/image-20210404001909534.png)
上图所示：查找Windows用户帐户名。
根据IDS警报，这很可能是Dridex感染。警报列表以以下几种警报结尾：ET TROJAN ABUSE.CH SSL Blacklist Malicious SSL certificate detected (Dridex)

Shown above: Alerts from Sguil on Security Onion using the EmergingThreats (ET) ruleset.

上图所示：使用EmergingThreats（ET）规则集的来自Sguil的关于安全性洋葱的警报。

这与我见过的其他Dridex感染相吻合。在这些Dridex警报之前，有条目指示Windows EXE或DLL文件是通过TCP端口8000从91.121.30.159下载的。可以通过使用菜单路径文件→导出对象→HTTP从此pcap导出此可执行文件，如下图所示。 。
如上所示：使用Wireshark从HTTP通信中导出对象。
上图所示：从HTTP流量中导出actiV.bin。
在导出文件之前，如何确定这是Windows可执行文件？遵循该HTTP请求的TCP流，它显示服务器返回可执行文件。 Windows可执行文件的指示符如下图所示。
![image-20210404002053645](/Users/yueting/Library/Application Support/typora-user-images/image-20210404002053645.png)
上图所示：跟随HTTP请求的TCP流到达91.121.30.169。
上图所示：您正在处理EXE或DLL文件的指示器。
从pcap中提取EXE后，可以将其提交给VirusTotal以获取有关它的更多信息，例如SHA256哈希。 VirusTotal链接：这里。
