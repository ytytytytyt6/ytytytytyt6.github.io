TASK

  • Write an incident report based on the pcap and the alerts.

  • The incident report should contains 3 sections:

  • Executive Summary: State in simple, direct terms what happened (when, who, what).
  • Details: Details of the victim (hostname, IP address, MAC address, Windows user account name).
  • Indicators of Compromise (IOCs): SHA256 hashes and details of the malware and/or artifacts, IP addresses, domains and URLs associated with the infection.

笔记
AgentTesla是一个信息窃取者。 AgentTesla有两类：
•使用SMTP
•使用FTP
从这种感染中看到的AgentTesla是使用SMTP的类型。即使通过TCP端口587（通常用于加密的SMTP），流量也不会被加密。
您可以使用文件->导出对象-> IMF来导出来自pap的通过SMTP发送的5封电子邮件。
在发出的5封电子邮件中有4个主题行：

![image-20210406232659027](/Users/yueting/Library/Application Support/typora-user-images/image-20210406232659027.png)•主题：PW_ronaldo.paccione / DESKTOP-M1JC4XX
•主题：CO_ronaldo.paccione / DESKTOP-M1JC4XX
•主题：SC_ronaldo.paccione / DESKTOP-M1JC4XX（发送两次）
•主题：KL_ronaldo.paccione / DESKTOP-M1JC4XX
包含两个字母的前缀如下：
•PW-在受感染主机上找到的密码。
•CO-来自网络浏览器的cookie数据
•SC-屏幕截图，带有jpeg附件
•KL-键盘记录程序数据
从pcap导出这些电子邮件，并在文本编辑器或Thunderbird之类的电子邮件客户端中查看它们。如果这是现实生活中的情况，则可以向受感染的用户确切显示从他们的计算机上偷来的东西。
许多AgentTesla恶意软件都使用加密的SMTP，因此我们很幸运，该恶意软件使用了未加密的SMTP，由于存在警报，因此更易于识别：
ET TROJAN AgentTesla Exfil Via SMTP\
从UTC 22:43开始获得58次点击
域paste.nrecom.net与pcap中的EXE相关联。我们看不到URL是什么，因为它是HTTPS。
但是到此域的HTTPS流量由初始EXE调用，我们将在其中看到类似以下内容的信息：
https://paste.nrecom.net/view/raw/b44fe71a
上面的代码在我检查时返回了base64编码的十六进制字符串。我将十六进制字符串转换为二进制，在上述情况下，我对每个字符串与0x02进行了XOR运算，以查找似乎是恶意的EXE。
值得注意的是，我认为域paste.nrecom.net本质上不是恶意的。这只是AgentTesla的恶意软件作者在过去一周左右的时间内使用的另一种类似Pastebin的服务。
